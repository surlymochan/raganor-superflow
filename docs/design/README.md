## 系统架构

### 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                     Superflow 工作流                         │
├─────────────────────────────────────────────────────────────┤
│  阶段 1: 脑暴 → 阶段 2: 计划 → 阶段 3: 执行 → 阶段 4: 归档     │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                    OpenSpec 规范驱动                         │
├─────────────────────────────────────────────────────────────┤
│  specs/ (真理源)  ←→  changes/ (变更提案)  →  archive/      │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                   GitHub 质量门禁                            │
├─────────────────────────────────────────────────────────────┤
│  CI → Code Review → Adversarial QA → CodeQL → Labeler      │
└─────────────────────────────────────────────────────────────┘
```

## 工作流设计

### 四阶段门禁

| 阶段 | 命令 | 输入 | 输出 | 门禁 |
|------|------|------|------|------|
| 1. 脑暴 | `/superflow:brainstorm` | 一句话需求 | 设计文档 + Spec 草稿 | 用户批准 |
| 2. 计划 | `/superflow:plan` | 批准的设计 | Spec 定稿 + 任务清单 | 用户批准 |
| 3. 执行 | `/superflow:execute` | 批准的任务 | 可工作的代码 | 测试通过 |
| 4. 归档 | `/superflow:archive` | 完成的实现 | 归档文档 | Challenge + 用户验收 |

### 执行模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| A | 任务独立、快速迭代 | 子代理驱动，自动审查 |
| B | 需要人工检查点 | 批次执行，干净上下文 |

## GitHub 规范

### 核心工作流（必须）

- `ci.yml` - CI/CD
- `publish.yml` - NPM 发布
- `code-review.yml` - 代码质量门禁

### 增强工作流（推荐）

- `qa-adversarial-review.yml` - 对抗性 QA 审查
- `codeql.yml` - CodeQL 安全扫描
- `labeler.yml` - PR 自动打标签

### 自动化工作流（可选）

- `auto-response.yml` - PR 自动回应
- `stale.yml` - 自动关闭 stale issue
